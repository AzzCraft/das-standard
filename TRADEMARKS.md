# Trademark and Brand Usage Policy

This repository defines the **DAS: Docs as Software Standard** (中文：文码合一标准). The community benefits when the name and brand communicate a consistent meaning.

This policy is intended to:
- help users describe compatibility accurately,
- prevent confusing or misleading usage of the DAS name, and
- protect the integrity of “DAS-compliant” claims.

> This document is provided for clarity and community governance. It is not legal advice.

## What names this policy covers

The following names are used to identify the standard and the canonical specification:
- **DAS**
- **Docs as Software**
- **DAS Standard**

AzzCraft Inc.  may (now or in the future) register one or more marks for these names or related logos.

## Allowed uses (nominative / descriptive)

You may use the names above to:
- refer to the standard in documentation or presentations, for example, “implements DAS Standard v1.4.6”,
- link to this repository,
- compare DAS to other approaches,
- state that your project is “compatible with” or “based on” DAS (see the compliance rules below).

## “DAS-compliant” and compliance claims

You may claim **“DAS-compliant”** (or similar language such as “DAS-compatible”) only if:
1. You clearly state *which* DAS version you target (e.g., “DAS Standard v1.4.6”), and
2. Your project passes the **Official Reference Validator** (`das-cli`) for that version without errors, and
3. You do not imply endorsement by AzzCraft unless you have written permission.

Recommended compliance phrasing:
- “DAS Standard v1.4.6 — validated by the reference validator for that version”

## Restricted uses

Without prior written permission, you may not:
- use “DAS” or “Docs as Software” as the primary brand for a competing standard or commercial product (e.g., you cannot name your product "The DAS Server"),
- Use the name in a way that implies official endorsement, certification, or affiliation with AzzCraft Inc.
- use a confusingly similar name or logo for a product/service that is unrelated to the standard.

## Logos

If a logo is published in this repository, you may use it only to reference the standard.
You may not use the logo as the primary identifier for your own product or company.

## Reporting misuse / requesting permission

If you believe someone is misusing the name or making misleading compliance claims, open an issue in this repository.

If you want to use the name or logo in a way that could be confusing (for example, in a product name or domain name), please request permission via a GitHub issue.
